# Realtime Notifications Platform 🚀

Полнофункциональная демо-платформа уведомлений реального времени, демонстрирующая 4 ключевых протокола веб-взаимодействия между клиентом и сервером.

Проект построен по архитектуре **монорепозитория (`npm workspaces`)** с общим пакетом типов, стильным интерфейсом на базе **React 19**, **Ant Design** и NestJS бэкендом.

---

## 🛠 Протоколы связи

Приложение позволяет на лету переключаться между 4 популярными техниками передачи данных в реальном времени:

1. **Short Polling (Короткие опросы)**  
   Регулярные HTTP GET запросы с интервалом в 5 секунд для получения обновлений.
2. **Long Polling (Длинные опросы)**  
   Сервер удерживает HTTP-соединение открытым до появления нового уведомления или таймаута (20 сек).
3. **SSE (Server-Sent Events)**  
   Постоянный однонаправленный HTTP-поток событий от сервера к клиенту (`EventSource` / RxJS).
4. **WebSockets (Socket.io)**  
   Полнодуплексный двусторонний канал реального времени. Поддерживает как мгновенное получение серверных событий, так и отправку пользовательских сообщений.

## 🎨 Ключевые возможности интерфейса

- **Светлая и Тёмная темы**: Переключение оформлений в один клик с автоматической подстройкой цветовой палитры компонентов Ant Design (`ConfigProvider`).
- **Лента уведомлений в реальном времени**:
  - Мгновенный поиск по тексту с задержкой ввода (**Debounce 300ms**) во избежание лишних ререндеров.
  - Фильтрация по типу важности (`Инфо`, `Успех`, `Предупреждение`, `Ошибка`) и источнику (`Сервер`, `Клиент`).
  - Сворачиваемый аккордеон (`Collapse`).
- **Лог сетевой активности**:
  - Подробная консоль всех сетевых запросов и веб-сокет событий.
  - Сворачиваемый аккордеон (`Collapse`) для экономии рабочего пространства.
- **Индикация статуса и Latency**: Измерение задержки соединения в миллисекундах и визуальный индикатор состояния (`Подключено`, `Подключение...`, `Отключено`).
- **Форма отправки событий**: Интерактивная публикация пользовательских сообщений через WebSocket канал.

## 🚀 Быстрый запуск

### 1. Установка зависимостей

Установите зависимости для всех воркспейсов из корня проекта:

```bash
npm install
```

### 2. Настройка файлов переменных окружения (`.env`)

Создайте конфигурационный файл `.env` на основе имеющихся примеров:

```bash
# Скопировать пример для корня
cp .env.example .env

# Скопировать примеры для сервисов
cp frontend/.env.example frontend/.env
cp backend/.env.example backend/.env
```

Основные параметры конфигурации:

- `PORT` — порт NestJS бэкенд сервера (по умолчанию `3001`).
- `VITE_API_TARGET` — адрес бэкенд сервера для dev-проксирования Vite (по умолчанию `http://127.0.0.1:3001`).
- `VITE_API_URL` — базовый URL API бэкенда. При пустом значении использует автоматическое проксирование Vite (`/api` и `/socket.io`).

### 3. Запуск приложения в режиме разработки

Запустите сервер (NestJS) и клиент (Vite) параллельно из корня одной командой:

```bash
npm run dev
```

Сетевые порты приложения по умолчанию:

- **Frontend App**: порт `5173`
- **Backend API**: порт `3001`

### 4. Дополнительные команды

- **Запуск только бэкенда**:
  ```bash
  npm run dev:backend
  ```
- **Запуск только фронтенда**:
  ```bash
  npm run dev:frontend
  ```
- **Полная сборка всех воркспейсов**:
  ```bash
  npm run build
  ```
