# Realtime Notification Center ⚡️

Интерактивный стенд для наглядного сравнения 4 основных способов передачи данных в реальном времени: **Short Polling**, **Long Polling**, **SSE** и **WebSockets**.

---

## 🔍 Что внутри

В приложении можно переключать протоколы на лету и следить, как меняется поведение сетевого соединения:

- 🔄 **Short Polling** — классический опрос сервера каждые 5 секунд. Простой подход, но дает лишнюю нагрузку и задержку между обновлениями.
- ⏳ **Long Polling** — клиент делает запрос и «ждет», пока сервер не пришлет новое событие или не выйдет таймаут (20 секунд).
- 📡 **Server-Sent Events (SSE)** — однонаправленный постоянный поток от сервера к клиенту поверх HTTP. Отлично подходит для лент новостей и стандартных уведомлений.
- ⚡ **WebSockets (Socket.IO)** — полноценный двусторонний дуплексный канал. Позволяет не только мгновенно получать события, но и отправлять свои прямо из интерфейса.

---

## 🎨 Фичи интерфейса

- 🌓 **Темная и светлая тема** — переключение оформления в один клик.
- 🔍 **Поиск и фильтрация** — удобный поиск по тексту уведомления (с дебаунсом 300мс) и фильтры по типам (`Info`, `Success`, `Warning`, `Error`).
- 📊 **Лог сетевой активности** — видна вся внутренняя кухня: типы запросов, статусы подключений и задержка (latency) в миллисекундах.
- ✍️ **Отправка пользовательских событий** — форма для отправки кастомных сообщений через WebSocket.

## 🛠 Технологии

Проект собран в виде монорепозитория (`npm workspaces`):

- **Frontend**: React 19, Vite, Ant Design, Socket.io Client
- **Backend**: NestJS, RxJS, Socket.io
- **Types**: общий пакет `@packages/types` с интерфейсами типов данных для клиента и сервера

## 🚀 Быстрый запуск

### 1. Установка зависимостей

```bash
npm install
```

### 2. Настройка файлов окружения (`.env`)

Создайте конфигурационные файлы из шаблонов:

```bash
cp .env.example .env
cp frontend/.env.example frontend/.env
cp backend/.env.example backend/.env
```

### 3. Запуск приложения

Запустите бэкенд и фронтенд одной командой из корня:

```bash
npm run dev
```

Приложение доступно по адресам:

- **Frontend**: http://localhost:5173
- **Backend**: http://localhost:3001

---

## 💡 Полезные команды

- `npm run dev:backend` — запуск только бэкенда (NestJS)
- `npm run dev:frontend` — запуск только фронтенда (Vite + React)
- `npm run build` — полная сборка всех воркспейсов
- `npm run lint` — проверка линтером
